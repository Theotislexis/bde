Updating Release Notes¶

These instructions assume you’re writing release notes for BDE 3.102.0.

When adding a new release notes page:

  • create a BDE-3.102.0-release-notes branch off of master

  • add the new release to the top of the list in docs/releases_notes/release_notes.rst

  • add in a blurb to the top of docs/index.rst (by copy/pasting the previous top release note and updating the version #’s and the date - note that the version number is in the blurb title AND in the link)

  • add a new docs/release_notes/BDE_3.102.0.rst file, by copying BDE_3.101.0.rst and fixing the version numbers, dates, drqs list, and highlights. Do not forget to add the new file to your git commit.

The way to generate the DRQS list is to run {WSCH /gr 101,2217 /qu 3.102.0<GO>}, copy the results from the terminal screen, and then paste them into this command, not forgetting the version argument:

$ bde-docs/bin/generateDrqsList.sh 3.102.0  # paste then hit ^D

This generates everything you need to paste into the bottom of the file, replacing the old “{{{ internal” and “{{{ oss” blocks of tickets from the previous release.

Note that the oss DRQS list should be edited to remove any references to repos other than bde-tools or bde - none of the changes in other repos are visible outside of Bloomberg.
